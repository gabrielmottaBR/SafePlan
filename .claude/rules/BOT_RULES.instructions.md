Essas instruções serão aplicadas para garantir a estutura do projeto e manter a consistência do código. Siga as diretrizes abaixo ao gerar ou revisar código:
1) Mantenha a estrutra de pastas organizada e divida por funcionalidades (ex: src/data, src/sensors, app/pages, etc.) e evite criar arquivos soltos na raiz. Também mantenha os arquivos de teste organizados em uma pasta `tests/` correspondente. Além de manter a separação de backend e frontend, evite misturar código de diferentes camadas.
2) Não é necessário criar arquivos md ou de documentação para cada funcionalidade, a documentação deve ser centralizada em um arquivo `docs/` e referenciada no README. Evite criar arquivos de documentação redundantes ou desnecessários.
3) Siga as convenções de nomenclatura do Python para arquivos, classes, funções e variáveis. Use snake_case para arquivos e funções, PascalCase para classes, e evite abreviações confusas. Mantenha a consistência na nomenclatura em todo o projeto.
4) Mantenha o código limpo e legível, seguindo as melhores práticas de Python. Evite código complexo ou difícil de entender, e prefira soluções simples e diretas. Use comentários para explicar partes complexas do código, mas evite comentários óbvios ou redundantes.
5) Garanta a modularidade do código, dividindo funcionalidades em funções e classes bem definidas. Evite funções ou classes muito longas ou com muitas responsabilidades. Mantenha a coesão dentro de cada módulo e minimize o acoplamento entre eles.
6) Siga as melhores práticas de segurança ao lidar com dados sensíveis, como credenciais
7) Garanta que ao final de uma solicitação o código está totalmente executável e sem erros, evitando deixar código incompleto ou com erros de sintaxe. Se necessário, adicione placeholders ou comentários indicando onde o código precisa ser completado.
8) Faça o uso racional do agente de conversação, evitando solicitar que ele gere grandes blocos de código de uma só vez. Divida as tarefas em partes menores e solicite a geração de código passo a passo, garantindo que cada parte seja revisada e corrigida antes de prosseguir para a próxima.
